Multi-Merchant Agentic Commerce

A production-shaped reference implementation of the protocols behind the Universal Shopping Cart — UCP, A2A, and AP2 — with two engineering deep dives where the hard problems actually live.

UCP · Universal Commerce Protocol A2A · Agent2Agent AP2 · Agent Payments TypeScript MIT

When an autonomous shopping agent fills one basket from many independent merchants, two problems dominate the engineering: how do you check out atomically across systems you don't control, and how do agents authenticate and authorize each other safely? This codebase answers both with running TypeScript, then surrounds them with the supporting protocol plumbing.

System architecture

A shopping agent discovers and verifies each merchant agent over A2A, reads their UCP catalogs, aggregates a Universal Cart, then drives a single orchestrated checkout — guarded by the auth gateway and settled against an AP2 payment mandate.

Quick start

git clone https://github.com/arjun-basu001/bucket-no-more.git
cd bucket-no-more
npm install

npm run typecheck      # strict TypeScript
npm run demo           # full end-to-end flow
npm run demo:checkout  # happy path + rollback
npm run demo:auth      # JWT, OAuth2, rate limiting, card signatures
Tip — every module is dependency-light and framework-agnostic. The seams (MerchantAdapter, StateStore, BucketStore) are interfaces you can back with Postgres, Redis, or your existing services.
