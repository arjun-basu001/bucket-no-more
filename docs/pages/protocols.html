PROTOCOLS

UCP · A2A · Agent Cards

The supporting plumbing that lets a shopping agent discover merchants, read their catalogs, and speak a common language.

📦 UCP — Universal Commerce Protocol

UCP defines a unified product schema so a shopping agent can read catalogs from many merchants through one shape, regardless of each merchant's internal data model. A merchant agent exposes its catalog as UCP products; the shopping agent queries and composes them into a Universal Cart.

const catalog = new UCPCatalog('audiophile');
catalog.upsert(normalizeToUCP(rawMerchantPayload, 'audiophile'));

const results = catalog.search({ text: 'headphones', maxPriceMinor: 30000, inStockOnly: true });
const lineItem = catalog.toLineItem('sku-aurora-headphones', 1);  // → Universal Cart line

The schema is validated with zod (fail fast, friendly errors) and supports money in integer minor units, availability states, variants/SKUs, categories, and a passthrough attributes bag for merchant-specific fields.

flowchart LR
    M1["Merchant A payload"] --> N["normalizeToUCP()"]
    M2["Merchant B payload"] --> N
    N --> P["UCPProduct (validated)"]
    P --> CAT["UCPCatalog.search / toLineItem"]
    CAT --> CART["Universal Cart line items"]
Heterogeneous merchant data → one validated UCP shape.

🪪 Agent Cards

An Agent Card is the A2A discovery document: a signed JSON object an agent publishes (typically at /.well-known/agent-card.json) so peers can learn who it is, where to reach it, what it can do, and how to authenticate to it.

{
  "schemaVersion": "1.0",
  "id": "agent://fleetfoot/merchant-agent",
  "name": "FleetFoot Merchant Agent",
  "endpoint": "https://api.fleetfoot.example/a2a",
  "provider": { "name": "FleetFoot Inc." },
  "auth": {
    "type": "oauth2-client-credentials",
    "tokenEndpoint": "https://auth.fleetfoot.example/token",
    "jwksUri": "https://auth.fleetfoot.example/jwks",
    "scopes": ["catalog:read", "checkout:execute"]
  },
  "capabilities": [
    { "id": "search", "name": "Catalog", "method": "ucp.catalog.search" },
    { "id": "reserve", "name": "Checkout", "method": "checkout.reserve" }
  ],
  "proof": { "type": "JsonWebSignature2020", "verificationMethod": "...#key-1", "jws": "..." }
}

Cards are created, signed, and validated through createSignedAgentCard() / verifySignedAgentCard(). Structure is validated with zod; the signature uses the canonical-JSON detached JWS scheme covered in the Auth deep dive.

🤝 A2A — the trust handshake

Before two agents transact, they perform a discovery + trust handshake. The output is a Session used to make authenticated A2A method calls (JSON-RPC 2.0 envelopes).

  1. Discover — fetch the responder's signed Agent Card (from its well-known URL).
  2. Verify — validate the card signature against the responder's published key, anchoring trust.
  3. Negotiate — agree on a protocol version and the intersection of desired vs. advertised capabilities.
  4. Authenticate — obtain an OAuth2 access token scoped to the responder's audience.
sequenceDiagram
    participant SA as Shopping Agent
    participant MA as Merchant Agent
    participant AS as Auth Server
    SA->>MA: GET /.well-known/agent-card.json
    MA-->>SA: signed Agent Card
    SA->>SA: verify card signature (trust anchored)
    SA->>SA: negotiate version + capabilities
    SA->>AS: client_credentials (audience = merchant API)
    AS-->>SA: access token
    Note over SA,MA: Session established → JSON-RPC method calls
Discover → Verify → Negotiate → Authenticate.
const session = await performHandshake({
  responderCard: merchantCard,
  responderPublicKey: merchantCardKey,
  desiredMethods: ['ucp.catalog.search', 'checkout.reserve'],
  oauthClient,                              // bound to the merchant token endpoint
});

💳 AP2 — Agent Payment Protocol (in this codebase)

AP2 governs payment mandates: a user delegates spend authority (up to a cap) to a shopping agent, which the orchestrator presents when authorizing charges. In this reference, the mandate flows through CheckoutContext.paymentMandate and underpins the authorize→capture split — funds are held under the mandate and only captured on full-basket success.
